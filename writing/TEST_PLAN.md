# 自动化文章写作系统测试集方案

## 测试目标

验证首版本地 Web 应用的核心闭环：任务输入、Agent 编排、评分准入、低分回流、文件化恢复、平台差异化发布包、导出，以及缺少模型配置时的错误处理。

## 自动化测试集

1. 正常全流程通过
   - 输入完整任务、三平台、用户资料、开启搜索。
   - 期望状态为 `approved`，总分大于等于 8，生成公众号/小红书/今日头条三套发布包。

2. 低分回流
   - 使用包含“低分回流”的选题触发 mock 首轮扣分。
   - 期望首轮低于 8 分，系统生成 `repair-*` 产物，并在后续轮次通过。

3. 问题责任映射
   - 检查编辑问题能分派到对应写手 Agent。
   - 期望事实问题归张素材，合规问题归钱人味，标题/结构问题归赵立意。

4. 平台差异化
   - 只选择公众号运行。
   - 期望最终发布包只包含 `wechat`，并包含标题、摘要、标签、封面方案和图片提示词。

5. 文件化恢复与导出
   - 运行后重新读取任务状态文件。
   - 期望状态、产物和 `final-package.md` 可恢复，导出内容包含三平台章节。

6. 素材来源处理
   - 开启搜索但不配置 `SEARCH_API_KEY`。
   - 期望系统调用默认 Research Skill 检索层；公开源无结果时返回 `research-no-results`，不再使用 `about:blank`。

7. 缺少 LLM 配置
   - 关闭 mock 且不设置 `LLM_API_KEY`。
   - 期望任务进入 `needs_config`，返回明确配置错误。

8. 默认素材检索源
   - mock `fetch`，模拟 Wikipedia、HN 等公开源返回。
   - 期望素材来源进入 Agent 上下文，保留标题、摘要和 URL。
   - GitHub Search 与 Crossref 保留为可选源，不放入默认源，避免泛主题抓取低相关结果。

9. 无素材阻断
   - 不提供用户资料，检索源也没有返回有效结果。
   - 期望任务进入 `needs_material`，不生成立意、初稿、运营发布包。

10. 中文查询改写与相关性过滤
   - 输入“今年高考如何填志愿”这类含弱关键词的问题。
   - 期望系统自动扩展为年份 + 高考志愿填报相关查询，并过滤掉只命中“今年”等弱词的无关来源。

## 手动验收

1. 访问 `http://localhost:5174/`。
2. 创建任务并运行流水线。
3. 查看 Agent 产物、评分与回流、最终发布包。
4. 点击导出发布包，确认 Markdown 能打开。
5. 刷新页面，确认历史任务仍可读取。
